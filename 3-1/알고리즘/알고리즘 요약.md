	2. Strategy - 전략 짜기

	5. 구현

		인풋: E, n, K

알고리즘과 문제 분석
	정확성, 일한 양, 공간사용, 최적, 간단의 주제로 분석
	정확성은 증명을 해야하고, 스텝마다 분석함
	-알고리즘은 인풋이 아웃풋으로 변하는 스텝의 시퀀스로 구성됨
	-증명하기: 인풋이 충족되면 알고리즘이 종료될 때 아웃풋이 참이 됨
	.
	효율성 -> 일의 양/ 인풋의 크기에 의존
	.
	worst case 복잡도
	Dn(정의역) -> 인풋의 사이즈 n, I -> 정의역의 원소
	t(I) -> 기본 연산의 수
	W(n) = max{t(I) | ∊ Dn} -> 알고리즘의 워스트케이스 복잡도
	W(n)은 은 크기 n의 입력에 대해 알고리즘이 수행하는 최대 기본 연산 수

점근적인 증가율에 의한 함수 분류
	알고리즘의 수행시간
		-알고리즘의 시간 복잡도는 알고리즘의 각 단계에서 소모되는 기본 연산의 수라는 추정치를 사용해 알고리즘의 성능을 평가한 것을 가리킴
		-알고리즘의 수행 시간은 주어진 입력에 따라 달라짐 -> 입력의 개수 n별로 묶어서 생각
		-n이 충분히 클 때, f(n)이 점근적으로 어떤 함수와 비슷하게 증가하는지를 파악
		-빅오, 빅세타, 빅오메가, 리틀오, 리틀오메가 표기법은 이를 점근적 분석을 통해 체계적으로 분류해 표기하는 방법
	![[Pasted image 20250413024316.png]]
	n이 충분히 클 때 g(n)에 대한 f(n)의 비율이 일정 값 c를 초과할 수 없음
	![[Pasted image 20250413024516.png]]
	알고리즘의 수행시간을 점근적인 증가율로 분류할 때에는 Big-Theta 표기법을 사용하는게 가장 좋음
	(big-theta 표기법은 동치 관계를 정의하기 때문)
	-입력의 크기 n은 동일하더라도, 입력에 따라 알고리즘의 수행시간이 달라지는 경우가 있음
		이런 상황에서는 어떤 경우인지(최악(빅오), 최선(빅오메가), 평균(빅세타))명시해야 함
	기본적인 함수의 증가 차수
	로그함수 < 다항함수 < 지수함수(밑이 1보다 큰)
