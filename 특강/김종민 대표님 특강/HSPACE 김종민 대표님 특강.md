# 기업 보안
## 1. 기업 보안의 특징

수사 포렌식
- 사건 발생 시점부터 포렌식 시작
- 법적 효력이 중요
- 보안 기술과의 대립(비트락커 등)
- 아티팩트 수집 및 분석 기술 필요
기업 보안 포렌식
- 사건 발생 전부터 포렌식 시작
- 법적 효력 신경 안 씀
- 해킹 기술과 친밀(MitM)
- 데이터에 대한 이해를 바탕으로 설계 및 기획이 중요

거점 오피스, 재택 근무 등 -> 보안 리스크

제로 트러스트 관점 -> MFA 도입

출근, 컴퓨터를 키는 순간부터 모든 것을 감시

거의 북한..

문서 중앙화, 서비스 대부분 클라우드에 있음
-> 랜섬웨어 타격x (중소기업 뺴고)

문서도 트래킹 가능

네트워크, 망분리, 앤드포인트 모두 분리

온프로미스 시스템이 거의 없음 -> 로컬 LLM사용 쉽지않음
온프로미스 데이터 센터 등 -> 너무비쌈
클라우드 대부분 사용(AWS 등)

서비스
- 앱(클라이언트 사이드) -> 보호 로직, 로깅
- 웹(서버 사이드) -> 방어 로직, 로깅

안티치트를 아무리 잘해도 핵을 막을 수 없음..
-> 클라이언트 사이드라서
클라이언트 사이드는 해커가 뭐든 우회 할 수 있음
게임 핵을 막으려면 모든 계산을 서버 사이드에서 해야함.

자동차도 마찬가지. 소프트웨어 락은 모두 해킹 가능

결국 로그를 통해 비정상적 접근(해킹, 핵 등)을 잡아냄

**결론: 기업 보안에서 중요한 것 -> 도메인 지식**
해당 기업에서 보안의 역할이 어떤 것을 지키고, 어떤 일을 하는 것인지 잘 파악해야 함

보안은 trade-off -> 보안성을 높이면 편의성과 사용성이 떨어짐
보안을 잘하는게 중요x
대부분의 해킹은 당해도 ㄱㅊ -> 더 건강해질 수 있음
서비스만 안내려가면 됨....

고객들은 보안성을 가지고 제품을 쓰지 않음 -> 서비스가 더 중요

요즘엔 해킹? 자연재해로 봄.. -> 취약점이 LLM으로 인해 많이 나와서 오히려 보안 수요 증가

취약점 찾기나 버그 바운티 등은 모두 LLM이 함
중요한건 '마인드', 관심사

취약점이 있어도 고치는 비용이 더 많이 나오고 고치는게 더 불편하다? -> 안고침, 그냥 관리
관리적인 측면이 더 중요
