
## 자료구조
1. map
	연관 컨테이너, dictionary처럼 키와 값의 쌍을 저장하며, 키를 통해 값에 빠르게 접근할 수 있음
	### **핵심 특징**
	1. **키-값 쌍 (Pair)**: 모든 데이터는 키와 값의 쌍으로 저장됩니다.
	    - **키 (Key)**: 데이터를 검색하고 정렬하는 데 사용되는 고유한 식별자입니다. **중복이 허용되지 않습니다.**
	    - **값 (Value)**: 키에 연결된 실제 데이터입니다.
	2. **자동 정렬**: `std::map`에 원소를 삽입하면, 키를 기준으로 **항상 오름차순으로 자동 정렬**됩니다.
	3. **빠른 검색**: 원소를 검색, 삽입, 삭제하는 데 걸리는 시간 복잡도는 모두 **O(logN)**입니다. 이는 데이터가 많아져도 성능 저하가 크지 않다는 것을 의미합니다.
	선언 및 사용 예시
	`#include <map>
	`#include <string>`
	`// map<Key의 타입, Value의 타입>`
	`std::map<std::string, int> ages;` 
	`// 삽입 및 접근`
	`ages["Alice"] = 30;         // Alice라는 키에 30이라는 값을 저장`
	`ages["Bob"] = 25;`
	`std::cout << ages["Alice"]; // 키를 이용해 값을 바로 얻어옴`
2. 데크(deque)
	- 양쪽 끝에서 삽입과 삭제가 모두 효율적으로 일어날 수 있도록 설계된 자료구조, 양방향 큐의 줄임말(double-ended-queue)
	- 메모리 상에서 연속된 하나의 블록이 아니라, 작은 연속된 블록들의 배열로 구성됨, 이 블록들을 필요에 따라 동적으로 추가하거나 제가하면서 크기를 조절함
	- 벡터와 달리 메모리가 연속적이지 않기 때문에, 중간에 삽입/삭제하는 연산은 리스트보다는 느리고, 벡터보다는 빠르거나 비슷함
	양쪽 끝에서 빈번한 삽입/삭제가 필요할 때 사용
	
| **함수**                  | **기능**           |
| ----------------------- | ---------------- |
| **`push_front(value)`** | 덱의 **앞**에 요소를 추가 |
| **`pop_front()`**       | 덱의 **앞**의 요소를 제거 |
| **`push_back(value)`**  | 덱의 **뒤**에 요소를 추가 |
| **`pop_back()`**        | 덱의 **뒤**의 요소를 제거 |
| **`[i]` 또는 `at(i)`**    | 인덱스를 통한 요소 접근    |
